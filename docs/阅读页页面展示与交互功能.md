## 一、页面整体布局

阅读页以主文档为中心，两侧辅助功能按需出现：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                            顶部导航栏                                    │
│   [←返回]              会话名称(可点击编辑)           [☀/🌙主题] [📖专注] │
├──────────┬──────────────────────────────────────┬────────────────────────┤
│          │                                      │                        │
│  左侧栏   │                                      │    历史QA卡片展示区     │
│ (hover   │         主文档阅读区                   │   (75vw位置之后)        │
│  展开)   │        (10vw → 75vw)                  │                        │
│          │                                      │                        │
│ 知识卡片  │   ┌─章节导航─────────────────────     │   ╭──────────────────╮ │
│ 学习足迹  │   │ 第一章... 第二章...            │   │ │ QA节点列表        │ │
│          │   └───────────────────────────       │   │ │ ● Q: 解释...     │ │
│          │                                      │   │ │ ● Q: 总结...     │ │
│          │    Markdown + LaTeX 渲染内容          │   │ │ ● Q: 举例...     │ │
│          │                                      │   │ │ ...              │ │
│          │                                      │   │ ╰──────────────────╯ │
│          │    ┌──────────────┐                  │   ↑                     │
│          │    │选中文本矩形框│ ← QA交互面板      │   透明圆角矩形卡片       │
│          │    │  紧贴右侧    │                    │   紧贴段落右侧          │
│          │    └──────────────┘                  │                        │
├──────────┴──────────────────────────────────────┴────────────────────────┤
└─────────────────────────────────────────────────────────────────────────┘
```

**布局说明：**
- 主文档区占据屏幕中央黄金位置（约65vw宽度）
- 左侧栏默认隐藏，hover屏幕左边缘时滑出
- 右侧历史QA卡片区在75vw位置之后
- QA交互面板浮动显示，紧贴选区右侧
- 章节导航在文档左上角，最长不超过页面总长的30%

## 三、左侧边栏

### 展示方式

左侧边栏采用"隐形设计"——默认完全隐藏，需要时才出现：

```

│                                      │
│   主文档内容正常显示                  │
│   用户专注阅读                        │
│                                      │
└─────────────────────────────────────┘

Hover触发：鼠标移到屏幕左边缘8px区域
┌──────────┬───────────────────────────┐
│ ←←←←←←←  │                           │
│ hover触  │   主文档内容               │
│ 发区域   │                           │
│          │                           │
├──────────┤                           │
│          │                           │
│  左侧栏  │                           │
│  滑入    │                           │
│          │                           │
└──────────┴───────────────────────────┘

交互流程：
[鼠标移入左边缘触发区] → [左侧栏滑入展开]
       ↓
[鼠标移出左侧栏区域] → [延迟400ms] → [左侧栏滑出隐藏]
```

## 四、主文档阅读区

### 文档内容展示

主文档区是阅读核心，支持丰富内容渲染：

- Markdown完整语法（标题、段落、列表、表格、引用等）
- LaTeX数学公式（行内公式和块级公式）
- 代码块和引用块

**阅读体验设计：**
- 左边距10vw，为左侧栏预留触发空间
- 内容延伸到75vw，右侧留出QA卡片区
- 衬线字体增强阅读质感
- 行高1.8，段落间距适中

### 文本选择交互

用户选择文本时采用主题色矩形框高亮：

```
单行选区（宽度较窄）：
┌─────────────────────────────────────────────────────────────────────────┐
│   ...这是一段普通文本，用户选中了│这几个字│，继续后面的内容...              │
│                                 └──────┘                                 │
│                                  ↑                                       │
│                            矩形框高亮（主题色边框）                         │
└─────────────────────────────────────────────────────────────────────────┘

多行选区（默认全选整个区域）：
┌─────────────────────────────────────────────────────────────────────────┐
│   ┌──────────────────────────────────────────────────┐                  │
│   │ 第一行被选中的内容                                 │                  │
│   │ 第二行被选中的内容                                 │ ← 矩形框包裹    │
│   │ 第三行被选中的内容                                 │   整个多行区域  │
│   └──────────────────────────────────────────────────┘                  │
│                        ↑                                                  │
│                 矩形框高亮（主题色边框）                                    │
└─────────────────────────────────────────────────────────────────────────┘
```

**矩形框设计要点：**
- 使用主题色作为边框颜色
- 边框宽度适中，不遮挡文字
- 单行根据实际宽度显示矩形框
- 多行默认选中整个多行区域，矩形框完整包裹
- 选区以"专注"为目的，帮助用户聚焦关键内容

**选区交互流程：**
```
[用户拖拽选中文本] → [判断选区类型] → [显示矩形框高亮]
                                           ↓
                                    [检测选区位置]
                                           ↓
                                    [触发QA交互面板]
                                    面板紧贴选区右侧浮动
```

### 章节导航

章节导航以淡淡文字展示在文档左上角：

```
┌─────────────────────────────────────────────────────────────────────────┐
│   ┌─章节导航─────────────────────────────────────────────┐              │
│   │                                                        │              │
│   │  第一章 引言  ○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○ │              │
│   │  第二章 背景  ○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○○ │ ← 当前章节 │
│   │  第三章 方法  ─────────────────────────────────────────── │ ← 高亮    │
│   │  第四章 结果                                            │              │
│   │  第五章 总结                                            │              │
│   │  ...                                                    │              │
│   │                                                        │              │
│   │            [可滑动查看更多章节]                          │              │
│   │                                                        │              │
│   └────────────────────────────────────────────────────────┘              │
│   ↑                                                                       │
│   位于文档左上角，淡淡文字                                                 │
│   最长不超过页面总长的30%                                                  │
│   超出部分可滑动查看                                                       │
│                                                                          │
│   主文档内容...                                                           │
└─────────────────────────────────────────────────────────────────────────┘
```

**设计要点：**
- **位置**：文档左上角，作为文档的一部分而非独立悬浮
- **文字风格**：淡淡的灰色文字，不干扰阅读
- **透明度区分**：当前章节透明度较高（更明显），其他章节透明度较低（更淡）
- **高度限制**：最长不超过页面总长的30%
- **滑动交互**：超出部分可通过滑动查看更多章节

**交互行为：**
- 点击任意章节标题 → 滚动跳转到该章节位置
- 当前阅读章节自动高亮（透明度更高）
- 滚动页面时自动更新高亮章节
- 专注模式下收起隐藏

## 五、历史QA卡片展示区

### 概念区分：QA卡片 vs QA节点

**重要区分**：历史QA卡片和QA节点是两个不同层级的概念：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                          │
│   【历史QA卡片】                                                          │
│                                                                          │
│   定义：展示在右侧75vw位置的整个透明圆角矩形容器                           │
│   功能：承载某个段落的所有相关QA节点                                       │
│   触发：屏幕中心位置段落自动触发卡片展开                                   │
│   触发规则：同一时刻只显示一张卡片                                         │
│                                                                          │
│   【QA节点】                                                              │
│                                                                          │
│   定义：卡片内部的单个问答条目                                             │
│   内容：显示问题缩略文本，如"Q: 解释这段内容..."                           │
│   展示：一张卡片内可包含多个节点，全部展示                                 │
│   操作：点击节点可查看完整问答详情                                         │
│                                                                          │
│   ┌───────────────────────────────────────────────────────────────────┐ │
│   │                                                                    │ │
│   │   ╭──────────────────────────╮  ← 整个是【历史QA卡片】             │ │
│   │   │ ● Q: 解释这段内容...     │  ← 这是【QA节点1】                  │ │
│   │   │ ● Q: 总结核心要点...     │  ← 这是【QA节点2】                  │ │
│   │   │ ● Q: 举例说明...         │  ← 这是【QA节点3】                  │ │
│   │   │ ● Q: 这和之前的内容关联？│  ← 这是【QA节点4】                  │ │
│   │   │ ...                       │                                    │ │
│   │   │ (滚动查看更多...)         │  ← 超过10条提示                     │ │
│   │   ╰──────────────────────────╯                                     │ │
│   │                                                                    │ │
│   └───────────────────────────────────────────────────────────────────┘ │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 与QA交互面板的区别

这是**两种完全不同的功能**：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                          │
│   【历史QA卡片】                          【QA交互面板】                     │
│                                                                          │
│   功能：展示已有的问答历史               功能：发起新的问答                 │
│                                                                          │
│   触发：屏幕中心段落自动触发              触发：选中文本手动触发             │
│                                                                          │
│   位置：固定在75vw之后的右侧区            位置：浮动，紧贴选区右侧           │
│                                                                          │
│   内容：全部QA节点展示                    内容：当前问题的完整交互           │
│                                                                          │
│   操作：点击QA节点查看详情               操作：快捷提问/自定义追问/摘录     │
│                                                                          │
│   ┌──────────────┐                     ┌────────────────────┐            │
│   │ ╭──────────╮ │ ← 透明圆角矩形      │ 选中文本引用...     │            │
│   │ │ Q: 解释  │ │ ← QA节点列表        │                    │            │
│   │ │ Q: 总结  │ │                     │ [解释][总结][举例] │            │
│   │ │ Q: 举例  │ │                     │                    │            │
│   │ │ Q: 更多  │ │ ← 全部展示          │ 追问输入框...       │            │
│   │ │ ...      │ │                     │                    │            │
│   │ ╰──────────╯ │                     │ AI回答内容...       │            │
│   │  (>10条滚轮) │                     └────────────────────┘            │
│   └──────────────┘                                                       │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 历史QA卡片的触发机制

系统自动判断并展示当前屏幕中心位置段落相关的QA卡片：

```
[页面加载/滚动]
      ↓
[检测屏幕中心位置的段落]
      ↓
[查找该段落是否有QA记录]
      ↓
   ┌──┴──┐
   有QA  无QA
   ↓     ↓
[自动展开  [不显示]
 该段落    QA卡片
 QA卡片]
```

**触发规则：**
- **位置触发**：屏幕中心位置的段落自动触发QA卡片展开
- **只显示一张卡片**：同一时刻只显示一个段落的QA卡片
- **滚动更新**：滚动页面时，自动更新为新的中心段落QA卡片

**弃用的方式：**
- ❌ 不使用hover鼠标检测触发
- ❌ 不使用碰撞避让机制（只显示一张卡片，无需避让）

### QA节点的全部展示

QA卡片内的所有相关QA节点全部展示：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                          │
│   ╭──────────────────────────╮                                          │
│   │ ● Q: 这段内容是什么意思？ │ ← 所有QA节点全部展示                       │
│   │ ● Q: 请解释核心概念      │                                          │
│   │ ● Q: 总结一下要点        │                                          │
│   │ ● Q: 举例说明            │                                          │
│   │ ● Q: 这和之前的内容关联？ │                                          │
│   │ ● Q: 应用场景有哪些      │                                          │
│   │ ● Q: 如何理解这个公式    │                                          │
│   │ ● Q: 实际案例            │                                          │
│   │ ● Q: 扩展阅读            │                                          │
│   │ ● Q: 练习题              │ ← 第10条                                  │
│   │  ─────────────────────   │                                          │
│   │  (滚动查看更多...)        │ ← 超过10条提示滚轮翻页                     │
│   ╰──────────────────────────╯                                          │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

**展示规则：**
- 所有相关QA节点全部展示（不再限制为3个）
- 默认可见最多10条节点
- 超过10条时：
  - QA卡片内部支持鼠标滚轮翻页
  - 显示"(滚动查看更多...)"提示

**QA节点交互：**
- 点击任意QA节点 → 打开QA详情卡片查看完整问答
- hover时节点背景微微变亮

### QA详情卡片

点击QA节点可查看完整的问答详情（全屏遮罩层）：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         QA详情卡片 (全屏遮罩层)                           │
│                                                                          │
│   ┌──────────────────────────────────────────────────────────────────┐   │
│   │ 问答详情                                              [×关闭]    │   │
│   ├──────────────────────────────────────────────────────────────────┤   │
│   │                                                                    │   │
│   │ "选中的原文内容..."                                                │   │ ← 引用原文
│   │                                                                    │   │
│   │ AI回答的完整内容...                                                │   │ ← 回答内容
│   │ 支持Markdown和公式渲染...                                          │   │
│   │                                                                    │   │
│   │ ──────────────────────────────────────────────────────            │   │
│   │ [输入追问...]                          [追问]                      │   │ ← 追问输入
│   │                                                                    │   │
│   └──────────────────────────────────────────────────────────────────┘   │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

**卡片功能：**
- 显示完整的选区文本（引用格式）
- 显示AI回答的完整内容（支持公式渲染）
- 支持发起新的追问

**追问结果：**
追问完成后，新的问答会作为**独立的一级询问**保存，与原始问题平级：

```
追问流程：

[输入追问问题] → [提交] → [AI流式回答]
                          ↓
                   [保存为新QA节点]
                          ↓
                   [成为一级询问]
                          ↓
                   [在历史QA卡片中展示]

QA节点结构变化示例：

原始QA卡片：              追问后：

╭──────────╮            ╭──────────╮
│ Q: 原问题 │            │ Q: 原问题 │ ← 原始问题
│ Q: ...   │            │ Q: 追问1  │ ← 追问问题（新增，平级）

## 六、QA交互面板

### 功能定位

QA交互面板用于发起**新的问答交互**。用户选中文本后，面板出现，提供快捷提问和自定义追问功能。

### 触发时机与位置

```
[用户在文档中拖拽选中文本]
         ↓
[显示矩形框高亮选区]
         ↓
[释放鼠标完成选择]
         ↓
[系统检测选区矩形框位置]
         ↓
[QA交互面板紧贴选区右侧浮动显示]

**位置计算：**
- 面板以选区文字为准，紧贴选区矩形框右侧

## 十、交互动画与延迟设计

### 淡进淡出动画

所有辅助性UI元素的显示和隐藏都采用淡进淡出动画：

```
淡入动画 (出现)
opacity: 0 → 1
duration: 300ms
easing: ease-out

淡出动画 (消失)
opacity: 1 → 0
duration: 200ms
easing: ease-in
```

### 延迟时间设计

| 交互元素 | 延迟类型 | 延迟时间 | 说明 |
|----------|----------|----------|------|
| 左侧边栏 | hover触发 | 0ms | 立即响应 |
| 左侧边栏 | hover离开 | 400ms | 防误关闭 |
| 历史QA卡片 | 滚动切换 | 150ms | 防频繁切换 |
| 历史QA卡片 | 淡入 | 300ms | 平滑出现 |
| 历史QA卡片 | 淡出 | 200ms | 平滑消失 |
| QA交互面板 | 选区触发 | 0ms | 立即响应 |
| QA交互面板 | 滚动关闭 | 100ms | 微延迟关闭 |
| QA交互面板 | 淡入 | 300ms | 平滑出现 |
| QA交互面板 | 淡出 | 200ms | 平滑消失 |
| 章节导航 | 高亮切换 | 200ms | 平滑过渡 |
| 摘录功能 | hover展开 | 200ms | 文字滑出 |
| 快捷问题 | hover下划线 | 300ms | 动画展开 |

### 死区设置

**历史QA卡片 - 滚动死区：**
- 滚动距离 < 50px 时，不触发QA卡片切换
- 目的：防止微小滚动导致的频繁卡片切换

**左侧边栏 - hover死区：**
- 鼠标移出边栏后，延迟400ms才收起
- 目的：防止鼠标短暂移出导致边栏意外收起

**QA交互面板 - 滚动关闭死区：**
- 滚动距离 < 30px 时，不关闭面板
- 目的：阅读时轻微滚动不会意外关闭面板

**章节导航 - 高亮切换死区：**
- 段落中心距离屏幕中心 < 100px 时，才切换高亮
- 目的：防止章节高亮频繁跳动

### 设计原则

```
1. 立即响应 vs 延迟响应
- 用户主动触发的操作（选中文本、点击）：立即响应，0ms延迟
- 自动触发的操作（滚动切换）：延迟响应，避免频繁跳动
- 离开操作：延迟响应，防止误关闭

2. 淡入时间 vs 淡出时间
- 淡入时间较长（300ms）：让元素从容出现，不突兀
- 淡出时间较短（200ms）：让元素快速消失，不遮挡

3. 死区大小
- 滚动死区：50px，防止微小滚动触发
- 延迟死区：400ms，防止短暂离开触发
- 根据元素重要程度调整死区大小

4. easing函数
- 淡入：ease-out，开始快结束慢，自然落地
- 淡出：ease-in，开始慢结束快，快速消失
```
