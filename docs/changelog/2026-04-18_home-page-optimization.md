# 变更请求：主导航页 HOME 优化

**日期**：2026-04-18
**变更原因**：当前 Home.vue 页面存在滚动问题、会话访问功能不完善、整体功能缺失

---

## 变更内容

### 1. 问题分析

| 问题 | 描述 |
|------|------|
| 滚动失效 | `el-main` 区域无法正常滚动，内容超出时被截断 |
| 会话访问 | 点击"继续"按钮后无法正确恢复历史会话状态 |
| 功能缺失 | 缺少会话归档状态区分、缺少删除功能、缺少搜索功能 |
| UI 不一致 | 与 Read.vue 的 Notion 风格不统一 |

### 2. 新旧对比

| 功能 | 旧版 | 新版 |
|------|------|------|
| 页面滚动 | ❌ 无滚动 | ✅ 正常滚动 |
| 会话恢复 | ❌ 仅跳转 | ✅ 加载历史问答 |
| 会话状态 | ❌ 无区分 | ✅ draft/archived 分区 |
| 删除功能 | ❌ 无 | ✅ 删除文档/会话 |
| 主题一致 | ❌ Element 默认 | ✅ 与 Read.vue 统一 |

## 验收标准

### Given-When-Then

**V1: 滚动功能**
- Given: 用户进入 Home 页面
- When: 文档/会话列表超出屏幕高度
- Then: 页面可正常滚动，无内容截断

**V2: 会话恢复**
- Given: 存在历史会话
- When: 用户点击"继续阅读"
- Then: 正确跳转并加载历史问答记录

**V3: 状态区分**
- Given: 存在 draft 和 archived 会话
- When: 页面加载
- Then: 两类会话分区显示

**V4: 删除功能**
- Given: 用户有文档或会话
- When: 点击删除按钮
- Then: 确认后删除，列表更新

**V5: 开启新对话**
- Given: 用户点击"上传文件"
- When: 新会话 页面加载，开始流式输出页面
- Then: 会话列表显示新会话，且默认选中
